# Ticketing App

A ticketing system built as a **Power Apps Code App**, connecting to Microsoft Dataverse. The UI is inspired by Model-Driven Apps but built from scratch with React, Vite, and TypeScript вЂ” full control, no compromises.

**Live app:** `https://apps.powerapps.com/play/e/2e4ff4ce-5107-eaa2-b6dd-a2832dee7708/app/fd20a5ab-1f4a-413d-81cf-e40279926fe7`

---

## What It Does

- List, create, and delete support tickets stored in Dataverse
- Filter and sort tickets by title or created date
- Select multiple rows and bulk delete
- Click any row to open a slide-in detail panel
- Color-coded status badges (Open, In Progress, Resolved, Closed)
- Switch between Tickets and Statuses views via the sidebar

## Local Development

### Prerequisites

- Node.js LTS
- Power Platform CLI (`pac`) вЂ” available via the [VS Code Power Platform extension](https://marketplace.visualstudio.com/items?itemName=microsoft-IsvExpTools.powerplatform-vscode) or `dotnet tool install Microsoft.PowerApps.CLI.Tool`
- Access to the Power Platform environment with Code Apps enabled

### Setup

```bash
# 1. Install dependencies
cd ticketing-app
npm install

# 2. Authenticate with Power Platform
pac auth create --deviceCode
# Follow the device code flow at https://microsoft.com/devicelogin

# 3. Select the environment
pac env select --environment 2e4ff4ce-5107-eaa2-b6dd-a2832dee7708

# 4. Run locally
npm run dev
```

Open the **Local Play** URL that appears in the terminal. Use the same browser profile you authenticated with.

> **Note:** Chrome/Edge may block local network access. Grant permission when the browser prompts.

## Build & Deploy

```bash
npm run build
```

On success you'll get the Power Apps play URL. The `power.config.json` already contains the registered `appId` вЂ” no need to re-init.

## Re-generating the Data Source

If columns are added or the schema changes:

```bash
# Remove the old data source
pac code delete-data-source -a "shared_commondataserviceforapps" -ds "commondataserviceforapps"

# Re-add it
pac code add-data-source \
  -a "shared_commondataserviceforapps" \
  -c "<your-connection-id>"
```

> After re-generating, re-apply the fix for the invalid TypeScript parameter name: find `MSCRM.IncludeMipSensitivityLabel` and replace with `MSCRM_IncludeMipSensitivityLabel` (6 occurrences in `MicrosoftDataverseService.ts`). This is a known bug in the generated output.
